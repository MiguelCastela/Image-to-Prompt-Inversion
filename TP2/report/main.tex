\section{Introduction}
Given a fixed text-to-image generator and a set of target images that it produced, we wish to recover text prompts that, when rendered again with the same model and same configuration, reproduce images visually close to the targets. The generator is the Latent Consistency Model \texttt{SimianLuo/LCM\_Dreamshaper\_v7}, a distilled variant of the DreamShaper / Stable Diffusion family, run under a fully specified and frozen configuration (8 inference steps, guidance scale 8.0, $768\times768$, per-image seed encoded in the filename).

\noindent
\\
Because prompt quality is judged entirely by the fidelity of the rendered reconstruction, a good description that yields a divergent image can produce bad metric scores, whereas a prompt that accurately reproduces the target image succeeds. Because the generator is deterministic, this problem is an optimisation in text space with a fitness function that lives in pixel space. This reframing motivated our methodology as the pipeline combines two methods of the approach spectrum: a vision–language captioning step that gives each target a deterministic initial prompt, and a closed-loop VLM-based refinement that renders that prompt, measures how far the render is from the target, and asks a vision–language model to fix the specific errors it sees, iterating until the reconstruction stops improving. An unused open-loop alternative that samples without rendering is briefly discussed in Section~\ref{sec:openloop}. The system is evaluated using three image-side metrics: CLIP image–image cosine similarity, LPIPS perceptual distance, and pixel MSE. Because these do not always agree, we also combine them into a composite rank (a scale-free rank-average) that we use to order candidates and select each target's top-3.

\section{Description of the problem}
\noindent
The test set consists of six target images (Figure~\ref{fig:targets}) generated by the fixed LCM and the prompts that produced them are not provided. The generator is deterministic given its inputs and the checkpoint, scheduler, step count, guidance scale, resolution, and the per-image rendering seed are fixed. The LCM render seed is fixed per image (the filename seed), so the renderer is deterministic and only the prompt search varies; the search is therefore repeated over five optimiser seeds ($1234$, $5678$, $9012$, $3456$, $7890$) with the render seed held fixed, and we report the mean~$\pm$~std across them.

\noindent
\\
With this, the search space (natural-language prompts) is discrete and unbounded, so we must rely on strong priors to obtain a viable starting point for the LLM-refining, rather than searching from scratch. Furthermore, because our prompt search isstochastic ( sampled from a language model), the protocol fixes the LCM rendering seed across all repetitions and repeats the search across multiple optimiser seeds, reporting variability.

\section{Methodology and Approach}
The final methodology is a prompt refining pipeline that searches over candidate prompts by scoring the images generated by them directly against the target images. It has two stages, the generation of an initial prompt per target, and a closed-loop image-side refinement that improves it, always followed by a rendering, scoring and ranking step. The refinement is seeded directly from the Stage~1 prompt, it renders that prompt and then iterates on it, with the same fixed generator and three metrics used throughout.

\subsection{Stage 1: Vision–language initial prompt}
Because the immense scale of word combinations makes blind text search unfeasible, we establish a strong initial prompt for each target using two complementary captioning models:

\paragraph{\normalfont\textbf{CLIP-Interrogator (style / medium).}} For each target image we make a single call to lip-interrogator's \texttt{interrogate()} routine, configured with the CLIP backbone \texttt{ViT-L-14/openai} (matching our evaluation encoder) and the \texttt{blip-large} captioner. Internally the routine combines two models with complementary roles. First, blip-large, an image-captioning model, looks at the image and writes a short natural-language description of its literal content, which pins down the subject but says little about artistic style, so to recover the style, the routine then turns to CLIP, a model that maps images and text into a shared embedding space in which cosine similarity measures how well a piece of text describes an image. The library also ships several curated lists of style phrases, its \emph{vocabulary banks}: artistic mediums (such as oil painting, 3d render, photograph), artist names, art movements, and a catch-all bank of aesthetic and quality descriptors (\emph{flavors}, such as ``octane render'' or ``volumetric lighting''). The target is embedded once with CLIP, every phrase in each bank is ranked by its similarity to that image embedding, and the single best-scoring phrase from each bank is appended to the blip-large caption. We take this string exactly as the library returns it, and store it once per image. Calling the library out of the box is a deliberate decision as an earlier attempt to score the vocabulary banks "by hand" produced skewed prompts, the story of which is told in Section~\ref{sec:failures}.

\paragraph{\normalfont\textbf{BLIP-2 (subject).}} We also run \texttt{Salesforce/blip2-opt-2.7b} steering it with a fixed instruction prompt (\texttt{``Question: What is the main subject of this image? Answer:''}). The reason for adding this model is that the clip-interrogator string is strong on style but unreliable on the subject as the blip-large caption states objecta with low fidelity. Since the subject is important in a text-to-image prompt, we want it stated cleanly and placed first, so that generation is anchored on the object. This step is skipped when the CLIP-Interrogator already opens with the same subject. We keep the two sources labelled rather than merged with the BLIP-2 phrase marked as the \textsc{subject} and the interrogator string as the \textsc{style / medium}. Giving them distinct roles stops the refinement model from blurring the two and drifting toward style at the expense of the subject. The resulting initial prompt is stored once per image and is the seed for the refinement loop.

\subsection{Stage 2: Closed-loop image-side refinement}
\label{sec:stage2}
 
For each target we render the Stage~1 initial prompt with \texttt{SimianLuo/LCM\_Dreamshaper\_v7} (the Latent Consistency Model), score it, then run a search in prompt space that, at every iteration, shows the vision–language model (\texttt{Qwen2.5-VL-7B-Instruct-AWQ}, an open weights $7$B-parameter multimodal LLM run locally in $4$-bit AWQ quantisation) both the \textbf{target image} and the \textbf{current best reconstruction}, together with that reconstruction's measured metrics, and asks it to propose improved prompts. The model can therefore react to its own rendering errors.
\noindent
\\
We ran the search with $12$ iterations. In each iteration the model is shown the branch's best reconstruction so far and asked for new prompts. These are all rendered, scored, and added to a growing candidate pool. The single highest-scoring prompt in that pool then becomes the ``current best'' that the next iteration is asked to improve on, while every prompt ever tried is held on an avoid-list so the model keeps refining the leader instead of circling back to weaker candidates. All images and scores are saved along the way as the submitted top-3 is selected from the whole accumulated pool at the end, not just from the last iteration. Forcing those proposals to be genuinely different from one another, and from earlier iterations, turned out to be one of the central problems of the projec. The mechanism that solves it is described in Section~\ref{sec:failures}.

\paragraph{\normalfont\textbf{What the model is given at each step:}} The initial prompt (Stage~1) is built from the target image alone. From then on, every refinement call to Qwen receives the same fixed group of inputs:
  \begin{enumerate}
    \item Two images, the target and the current best reconstruction, each downscaled to $512$\,px to fit GPU memory;
    \item The BLIP-2 and clip-interrogator outputs labelled (subject \& styling)
    \item The text of the current prompt, with a note that the second image is its render;
    \item The current similarity scores (CLIP, LPIPS and MSE of that render);
    \item The metric's one-line objective, worded so the model emphasises what that branch's metric actually rewards. For the \textbf{clip} branch: ``prioritise overall semantic and stylistic similarity to the target'', because CLIP measures semantic and stylistic agreement. For the \textbf{lpips} branch: ``prioritise perceptual similarity: textures, structures and mid-level detail'', the features LPIPS is sensitive to. For the \textbf{mse} branch: ``prioritise exact pixel match: get colours, brightness and the spatial layout precisely right'', which is what pixel MSE penalises. Finally, for the \textbf{composite} branch: ``balance semantic, perceptual and exact-pixel similarity to the target'', matching its blended objective.
    \item A single description ``axis'' to vary for this batch, that is, one aspect of the image (its colour palette, lighting, composition, medium, and so on) that the model is told to change more than the others, with the chosen axis rotating from one call to the next so successive batches diversify along different aspects (Section~\ref{par:eight});
    \item An avoid-list of the prompts already tried (the most recent twenty), with an explicit instruction not to repeat them.
\end{enumerate}
\noindent
When every prompt a call returns, after normalizing case and spacing, is already on the avoid-list, a stronger instruction is appended telling the model to re-describe the target from scratch (new sentence structure, synonyms, reordered tags) instead of lightly editing the current wording. Once enough distinct prompts have been collected, each is rendered under the fixed configuration and scored on the three metrics, and the branch then keeps, by its own metric, the single best-scoring candidate found so far as the ``current best'' that the next iteration works from. To avoid repeatedly generating prompts that have already been explored, both the proposal step and the refinement loop stop early when progress stalls. Within one iteration, the proposal step for a branch stops at whichever of these limits is reached first: it has collected eight distinct prompts (Section~\ref{par:eight}), five \emph{consecutive} calls have each failed to add a new prompt, or a hard cap of twelve calls to Qwen is hit. At the refinement level, if none of the four branches discovers a prompt or render that is new to the search, the image is considered converged and no further iterations are run for it. Generating a prompt is cheap next to rendering one, so the model gets many attempts to break out of repeating itself before a branch is stopped.                                                       

\paragraph{\normalfont\textbf{Four objective branches:}} 

The four branch-specific instructions described above are implemented as four parallel refinement branches. All four start from the same Stage~1 prompt, but each branch selects and propagates candidates using a different fitness function with CLIP maximizing CLIP image–image cosine similarity, LPIPS minimizing perceptual distance with an AlexNet backbone, MSE which minimizes pixel MSE in [0, 1] pixel space or the composite score which maximises a scale-free standardised score z(clip) $-$ z(lpips) $-$ z(mse), where z(·) is the z-score across the current pool. The branches therefore explore different regions of prompt space, and their candidates are pooled together before the final ranking.

\subsection{Rendering, scoring and ranking}
Every candidate prompt is rendered with the fixed LCM configuration and the per-image seed, then scored against the target with the four metrics. Rendering uses the negative prompt included with \texttt{clip-interrogator} (\texttt{negative.txt} with 40 terms such as blurry, low quality, deformed, oversaturated). The negative prompt lists undesirable image qualities and only affects generation when classifier-free guidance (CFG) is active. CFG uses both the positive and negative prompts to steer the denoising process toward the positive and away from the negative, with the strength of that steering controlled by the guidance scale. Because our guidance scale is fixed at $8.0$ (neutral value is $1$), CFG remains strongly active throughout the search, so the negative terms are consistently discouraged in every render.
 
\paragraph{\normalfont\textbf{Ranking.}}

When refinement finishes, each target has a pooled set of candidates from all four branches. To select the final top-3, we rank these candidates using a \textbf{Borda rank-average}.We therefore order the pool with a \textbf{Borda rank-average}. Within a target's pool of $N$ candidates, each candidate is ranked separately on each metric from $1$ (best) to $N$ (worst), where ``best'' means the \emph{highest} CLIP similarity but the \emph{lowest} LPIPS and the \emph{lowest} pixel MSE. Using ranks rather than raw metric values makes the score scale-free and prevents any single metric from dominating due to its numeric range. The candidates are then sorted by this score, and the top three become the submitted prompts for that target. This final ranking is independent of the branch that produced a candidate as the top-3 for a target may come from different branches or entirely from a single one. The best candidate under each metric is also shown in (Section~\ref{sec:permetric}).

\subsection{Challenges and design refinements}
\label{sec:failures}
Three concrete problems that dominated the development of this project:

\paragraph{\normalfont\textbf{Initial prompt:}}
\noindent
Our first attempt at the initial prompt construction consistently produced women or stylised female characters regardless of the true subject. This issue arose because instead of using clip-interrogator directly, we built a custom version that extracted its raw vocabulary files (artists, mediums, and aesthetic “flavors”), scored each entry with a zero-shot CLIP model and kept the ones with a high enough score (with 10 maximum), and combined the resulting rankings with structured BLIP-2 outputs (subject, lighting, medium, camera angle, and style).


\noindent
\\
This design introduced two compounding problems. First, BLIP-2 was queried using five structured VQA (Visual Question Answering) prompts (subject, lighting, medium, camera angle, and style in a Question: … Answer: format), but it only reliably answered the subject question with the remaining four frequently echoed the question or produced generic or incorrect responses, which significantly weakened the subject signal. Second, the vocabulary itself was strongly biased as some artist names such are heavily associated with stylised female portraiture in the training data, so their presence already biases generation toward women. This effect is further amplified by generic aesthetic terms such as beautiful, high quality, or masterpiece, which are strongly correlated with human faces and female subjects in large-scale text–image datasets and reinforcement-learning alignment data. When these tokens were ranked highly and passed to the VLM without a strong subject anchor, they shifted generation toward female characters even when the target image depicted something else.


\noindent
\\
We therefore removed the hand-built scoring entirely. The style component is now produced by a single \texttt{interrogate()} call, so the library scores its own vocabulary banks internally and returns one ready-made prompt string that joins the blip-large caption with the best-matching style phrase from each bank, while BLIP-2 is restricted to extracting only the subject, which is put first and labelled as \textsc{subject}. This separation ensures that the subject acts as an anchor during prompt construction, preventing stylistic tokens from dominating the output. With this change, the unintended drift toward female subjects disappeared.

\paragraph{\normalfont\textbf{Diversity across candidates:}}
\label{par:eight}
\noindent
A second challenge was ensuring diversity among candidates, given that the model tended to preserve the input prompt for many tries. Because the generator is deterministicS, any duplicated prompts produce identical renders and identical metric scores, so prompt variation had to be explicitly enforced. We first tried to spread the candidates apart with sampling alone, by raising the temperature and drawing each prompt independently, but both still collapsed onto the model's most likely continuation, which is to copy the prompt it was given. Passes like these produced as few as 2 distinct prompts out of 30. Given this, our decision was instead to generate the variations sequentially and condition each new prompt on the ones already produced with an \textbf{avoid-list} built into the refinement loop, as the model is shown every prompt already kept and every earlier proposal, and told to produce new ones that differ from all of them. 

\noindent
\\
For each target, at each iteration, and separately for each of the four metric branches, the proposal step loops the VLM until it has gathered \textbf{eight distinct prompts}, with five it intends to keep plus three spares (\texttt{N\_PROPOSALS}~=~$5$, \texttt{IMG\_RETRY\_BUFFER}~=~$3$). To further reduce collapse, we apply a \textbf{repetition penalty} ($1.2$), which discourages the model from reusing tokens it has already generated in the same output sequence. This directly targets a failure mode where, after producing a strong first prompt, the model tends to replicate it across the remaining numbered slots in the same call. This penalty reduces the probability of repeating previously generated tokens, thereby breaking this within-call cloning behaviour while still allowing necessary shared words such as the subject.

\noindent
Diversity is further encouraged through three additional mechanisms. First, a rotating description \textbf{axis} is used where each call is assigned a deterministic aspect (subject form, colour palette, lighting, composition, medium, or background) that it is instructed to vary, with the axis cycling in a fixed round-robin order across calls and shifting per iteration to avoid repetition patterns. Second, an \textbf{escalating temperature} is applied, so if a call fails to produce new prompts, the temperature is increased by $+0.15$ per consecutive failure up to a maximum of $1.20$. In addition, a conditional “diverge harder” instruction is appended to the user message (which contains all the per-call, changing state) when the proposal step stalls, explicitly pushing the model to abandon local paraphrasing and re-describe the target from scratch rather than making incremental edits to previous prompts. The unique prompts are then rendered and filtered at the image level, so any render that is effectively identical to one already kept this iteration render-level duplicates are filtered using a small dHash perceptual-hash distance (where exact matches are removed, corresponding to cases where different prompts produce identical renders, like for trailing or ignored tokens) and are discarded. The three \texttt{IMG\_RETRY\_BUFFER} spares refill these slot, so every branch still can land distinct reconstructions.

\noindent
Before settling on the current pipeline that passes the 6 initial pronts straight to the LCM, we explored an alternative where these initial prompts were expanded into 30 variations with the same VLM, rendered and the best kept. This never let the model see how its own prompts rendered, so it could not correct a specific mistake, which would make these initial renders worse that the ones in our current setup before refinment (explained at the beginning of (Section~\ref{sec:stage2}).

We evaluate the pipeline on the pool of \emph{surviving} candidates produced by the search. For each target, the closed loop runs four objective branches (CLIP, LPIPS, MSE, and the composite), and within each branch it keeps every candidate that was actually rendered and scored, discarding only exact prompt repeats and renders that collapse to a duplicate image under the dHash check. \subsection{Seeds and sampling temperature}
It is worth separating the two distinct sources of randomness in the pipeline, because they are handled in opposite ways.

\paragraph{\normalfont\textbf{Two kinds of seed.}} The first is the LCM \emph{render} seed, which controls the generator's own noise. It is fixed per image from the filename (for example \texttt{7836.png} renders with seed $7836$) and never changes, for any prompt, at any iteration, in any repetition of the experiment. This is what makes the generator deterministic, so that the same prompt always produces the same image and the metrics are a clean function of the prompt. The second is the \emph{optimiser} seed, which controls Qwen's stochastic prompt sampling. A global value (\texttt{set\_seed(1234)}) is set at start-up, but each individual proposal call also derives its own seed as \texttt{GEN\_SEED} plus a hash of the tuple (image, branch, iteration, call number). This keeps every call reproducible while making them mutually distinct: different branches and iterations do not replay the same sampling trajectory. Folding the iteration index into the seed is deliberate, since without it a new iteration would repeat the previous iteration's exact samples against a larger avoid-list, so every proposal would already be banned and the branch would collapse to zero new prompts. The optimiser seed is overridable from the environment, so the protocol's multi-seed sweep ($\ge 5$ repetitions) can vary the prompt search while the render seed stays fixed.

\paragraph{\normalfont\textbf{Temperature.}} Temperature applies only to Qwen; it does not touch the renderer, whose randomness is already pinned by the render seed (guidance scale and step count are constant). Proposals are sampled at a base temperature of $0.8$ with nucleus sampling (top-$p$ $0.95$), which gives varied but on-target wordings. When a proposal call returns nothing new, the temperature is escalated by $0.15$ per consecutive stale call up to a ceiling of $1.20$, flattening a peaked distribution until fresh wordings appear, and it drops back to the base once new prompts are found. A mild repetition penalty ($1.2$) further discourages the model from copying its own earlier output within a single call. The escalation only ever raises diversity: any malformed or off-target prompt that slips through at high temperature simply scores worse and is discarded during selection.

Two facts stand out. First, \textbf{CLIP similarity is essentially decoupled from the pixel/perceptual metrics}: it is mildly anti-correlated with LPIPS ($-0.268$) and effectively uncorrelated with pixel MSE ($-0.029$). A prompt can nail the target's semantics (high CLIP) while differing in layout and texture, so optimising CLIP alone would not minimise pixel error. Second, the two low-level metrics agree strongly ($+0.741$): a candidate close in raw pixels is usually close perceptually too. Together this justifies keeping the objectives separate and combining them only through the scale-free composite ranking, rather than collapsing everything into one number.

\subsection{Stability across seeds}
Figure~\ref{fig:seedstab} asks a different question from the tables: not how good the pool is, but whether any single seed is carrying it. For each seed in isolation we take its best-per-image value on each metric and average over the six targets; were the four searches unequal, one set of bars would stand out. They do not. The seeds fall in a narrow band on every metric (best CLIP $0.885$ to $0.893$, best LPIPS $0.471$ to $0.498$, best MSE $0.018$ to $0.020$), a spread far smaller than the across-target std within any one seed. We draw two conclusions. First, the pooled figures in Table~\ref{tab:aggregate} are robust rather than the product of one fortunate run, since each seed on its own reproduces almost the same peak quality. Second, this clarifies what the extra seeds actually contribute: they do not raise the ceiling (every search already reaches it), they enlarge the set of distinct strong prompts beneath it, which is precisely what allows each target to fill three genuinely different top-3 slots (Section~\ref{sec:limits}).

\subsection{Qualitative analysis: successes, failures, ambiguous cases}
\label{sec:limits}
\paragraph{\normalfont\textbf{Clear success, orange juice (1159\_25).}} The easiest target. Short, literal prompts reach CLIP up to $0.963$ and the lowest LPIPS of the set ($0.347$); the MSE-best render is closest in raw pixels (RMSE $0.129$). The subject is a single common photographable object with no stylistic ambiguity, so warm start and refinement converge quickly and all three objectives land on near-identical, faithful images.

\paragraph{\normalfont\textbf{High CLIP, high LPIPS, palm tree (1159\_29).}} The semantics are recovered very well (CLIP up to $0.935$) but LPIPS stays high (best only $0.606$) and MSE is among the worst of the set. The scene is correct in content (a palm tree on a rock in the ocean at sunset) but its exact composition (horizon line, wave pattern, sun position) is hard to pin down from text, so perceptual distance stays large despite strong semantic agreement. This is the clearest case of the metric disagreement quantified in Table~\ref{tab:spearman}.

\paragraph{\normalfont\textbf{Hardest semantic target, hedgehog (1159\_7).}} The lowest CLIP of the set: even the CLIP-best prompt reaches only $0.808$. The target is an out-of-distribution concept (a hedgehog of crystalline/translucent spikes on a foam cube) that the captioners struggle to name and the generator cannot reliably reproduce. This target also produced the clearest example of a \emph{gamed} metric: the LPIPS-best render came from the near-empty prompt ``Hedgehog, light amber coat, teen'' (CLIP only $0.529$), which scores well on LPIPS by being bland and low-detail while losing the subject entirely. It is a concrete argument against trusting any single low-level metric, and for keeping the composite ranker for the final selection.

\paragraph{\normalfont\textbf{Lowest absolute error, astronaut (7836).}} The lowest pixel error of the set (RMSE $0.101$, LPIPS down to $0.391$) with CLIP up to $0.917$, driven by the large dark, low-frequency background that is trivial to reproduce, a reminder that pixel MSE rewards background agreement as much as subject fidelity.

\paragraph{\normalfont\textbf{Diversity recovered by multi-seed pooling.}} In single-seed runs the aggressive image-level de-duplication collapsed easy targets (the rainbow hamster, the astronaut) onto near-identical prompts, so the submitted top-3 occasionally fell short of three distinct candidates. Pooling four independent optimiser seeds resolves this: because each seed explores a different sampling trajectory, the union supplies three genuinely distinct strong prompts for every target and all $18$ slots are filled (Annex~A, Table~\ref{tab:annex-top3}). The residual limitation is no longer diversity but metric gaming, as the hedgehog case shows, which is mitigated but not eliminated by the composite ranking.

\section{Conclusion}
We framed image-to-prompt inversion as text-space search under a fixed, deterministic LCM generator, scored entirely in pixel space, and built a pipeline that warm-starts from vision–language captioning (BLIP-2 subject $+$ CLIP-Interrogator style) and then refines that single prompt in a closed loop that reacts to its own renders through CLIP, LPIPS and pixel-MSE feedback across four objective branches.

The closed loop yields a strong best reconstruction per target (CLIP $0.849$, LPIPS $0.583$, pixel MSE $0.0288$ on average over the six images) and a tight candidate distribution, confirming the value of scoring in pixels at every iteration rather than trusting caption fluency. Its main weakness is diversity: on easy targets the image-level de-duplication collapses the loop onto near-identical prompts, so the submitted top-3 occasionally falls short of three distinct candidates. The metric analysis confirms that CLIP semantics and pixel/perceptual fidelity are only weakly correlated, validating the multi-objective design and the scale-free composite ranking. The whole pipeline is reproducible from Table~\ref{tab:repro}: every model version, hyperparameter and random seed is fixed, and the LCM rendering seed is held constant; the remaining step for full reproducibility is the multi-seed repetition of the stochastic search.
